DevLog 250528 — Digital Independence Declaration

> Log Date: 250518

I’m writing this directly from my freshly built Ubuntu forge — dual booted, RTX 3060 active, SSD whisper-silent. All systems go. Today I declare my digital independence.

My first project will be a video discussing a realization: we are living in a culture sculpted by data-mining algorithms, not people. Political bots shape the feed. Most information is cast into extreme left or right, while the middle — the place where mutual respect might live — gets filtered out. I don’t even want to see the hostility I see online, yet it’s forced into my viewport by sponsored content and AI-optimized psychological bait.

Human emotion is now a dataset. Anger sells. Fear converts. Context has vanished. In response, I’m leaning into self-hosted sovereignty. I want to build pipelines that parse facts from curated sources — not click-driven streams spoon-fed by Meta, Google, X, Bytedance or the ad auction of the day.

Stack Foundation

On this fresh Ubuntu install, I’ve configured:

  • NVIDIA drivers for RTX 3060 via ubuntu-drivers autoinstall
  • Docker, docker-compose, and Git
  • n8n automation engine
  • SSH keys and GitHub authentication

My IRC chat is connected across machines via a Web2 Netlify-hosted instance. This channel will become a nexus for automation feedback and system health as I grow the Forge.

Project: Automated NFT Creator

This first project will be a locally hosted automation pipeline using Docker + n8n, powered by models like Mistral and integrated with NFT.Storage and Polygon/ETH for minting. The aim is to:

  • Create a self-minting NFT engine
  • Use local LLMs to generate metadata and stories
  • Integrate facts and source validation through automated parsers
  • Trigger workflows via webhooks, IRC commands, or UI inputs

Everything will be pushed to GitHub and documented. I want this system to be usable by anyone exploring sovereignty through local tools and creativity.

Self-Hosted Philosophy

Whenever a local-first or open-source option exists, I will use it. Web3 tools are considered only if the alternative is centralization through a closed platform. I am rejecting the illusion of safety in SaaS, and replacing it with reliability in code, transparency in containers, and trust in my own hands.

— Lorelei Noble
