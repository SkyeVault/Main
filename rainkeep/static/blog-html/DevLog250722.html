DevLog 250722 — Full System + Git Remote Fix

> Log Date: 2025-07-22

What seemed like a slow month turned into a foundational shift in how I work—thanks to a rebuilt system, Nomachine setup, and Git remote hygiene.

Today’s work was about restoring access and rebuilding smarter. I reinstalled everything after a crash, and instead of just recovering, I upgraded: a fresh Ubuntu 24 setup, clean start script, and multi-device control over my GPU workstation.

Nomachine Workstation from the Kitchen Table

I can now remote into my tower from the kitchen using Nomachine. Moonlight failed due to macOS 12 incompatibility. Nomachine succeeded where others didn't, enabling access from both my MacBook and iPad. Blender sculpting with an Apple Pencil is now in scope.

Git Push + Submodule Fix

I ran into a broken submodule error from Netlify due to an old reference to stable-diffusion-v1. I removed it, cleaned up .gitmodules, and committed the changes. 

git rm --cached ArynCore/docker/stable-diffusion/stable-diffusion-v1
```

rm -rf .git/modules/ArynCore/docker/stable-diffusion/stable-diffusion-v1
rm .gitmodules
```

Pushed back to GitHub. Netlify builds now pass.

Check Git Remote URL

GitHub notified me that the repository location changed. To stop the redirect warnings, I updated the origin: 

git remote set-url origin https://github.com/SkyeVault/Main.git
```
```

Verified with:

git remote -v
```
```

Expected output:

origin  https://github.com/SkyeVault/Main.git (fetch)
```

origin  [https://github.com/SkyeVault/Main.git](https://github.com/SkyeVault/Main.git) (push)
```

That completes the cleanup. All repos point to the right place. Everything compiles. Blender is ready. This was a deep sync day.
